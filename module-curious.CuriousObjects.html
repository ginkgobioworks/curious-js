<static> parse(relationships, queryJSONResponse, existingObjects)

Get objects associated with each subquery. For each subquery, build a hash of ID to object. If existing objects are specified, will build relationships using the existing objects.
Parameters:
Name Type Description
relationships string[] The names of the relationships objects will have to one another.
queryJSONResponse Object An object of fields holding the query response, as returned and parsed directly from JSON without any post-processing.
Properties
Name Type Description
computed_on string The query timestamp.
last_model string The model name of the last set of objects returned.
results Object[] An array of objects containing Django object ids and other meta-information about the query; one element per model.
results[].model string The model name for this part of the query.
results[].join_index number The index of the model this model joins to.
results[].objects Array[] The IDs of the objects returned by the query
data Object[] An array of objects containing the other fields of the Django objects, more than just the IDs. See _parseObjects for a description of this data in the queryData parameter.
existingObjects Array.<Object.<number, Object>> The existing objects. Each object in the array is a mapping of an id to its corresponding object.
